# Gender Party Game

Локальная браузерная игра-викторина для гендер-пати. Один компьютер используется как экран администратора, а гости подключаются со смартфонов через общую Wi‑Fi-сеть.

Проект работает на Python, Flask, Flask-SocketIO и SQLite. Отдельный сервер или платный хостинг для домашней игры не требуется.

## Возможности

- игровое поле из 4 тем по 5 вопросов;
- вопросы с выбором варианта и вводом текста;
- несколько допустимых написаний правильного ответа;
- фото-карточки в формате JPG/JPEG;
- аукционный вопрос со ставками;
- рейтинг игроков и индикатор подключения;
- финальное голосование «Мальчик / Девочка»;
- автоматическое удвоение баллов угадавших игроков;
- секретный раунд с предложением и рейтингом имён;
- QR-код и ссылка для быстрого подключения гостей;
- полноэкранный режим для телевизора или проектора;
- полный сброс игры, включая игроков и результаты;
- работа в локальной сети без публикации сайта в интернете.

## Важное предупреждение

Проект рассчитан на доверенную домашнюю или офисную локальную сеть. Не открывайте порт игры в интернет и не настраивайте проброс порта на роутере.

Перед публикацией собственного набора вопросов проверяйте, нет ли в нём личных фотографий, фамилий, адресов, телефонов и других данных, которые вы не хотите сделать публичными.

## Требования

- Python 3.11, 3.12 или 3.13;
- Windows 10/11, macOS или Linux;
- компьютер и смартфоны, подключённые к одной Wi‑Fi-сети;
- современный браузер;
- интернет при первом открытии страниц игры: браузер загружает клиент Socket.IO с CDN.

Для Windows рекомендуется Python 3.12.

## Быстрый запуск на Windows

### 1. Скачайте проект

На странице репозитория нажмите **Code → Download ZIP**, распакуйте архив в обычную папку, например:

```text
C:\Games\gender-party-game
```

Не запускайте проект прямо из ZIP-архива.

### 2. Установите Python

Установите Python 3.11–3.13. Во время установки Windows включите пункт **Add Python to PATH**.

Проверка в PowerShell:

```powershell
python --version
```

или:

```powershell
py --version
```

### 3. Установите зависимости

Дважды щёлкните по файлу:

```text
setup.bat
```

Скрипт:

1. создаст виртуальное окружение `.venv`;
2. установит зависимости из `requirements.txt`;
3. создаст локальный файл настроек `.env`;
4. автоматически сгенерирует случайный `SECRET_KEY` и секретный адрес администратора.

### 4. Настройте игру

Откройте созданный файл `.env` в Блокноте. Секретный адрес администратора уже сгенерирован автоматически. Проверьте строку `ADMIN_SECRET_PATH` и настройте пол ребёнка:

```env
ACTUAL_GENDER=boy
```

Для девочки:

```env
ACTUAL_GENDER=girl
```

Сохраните файл.

### 5. Запустите игру

Дважды щёлкните:

```text
start_game.bat
```

Откроется отдельное окно сервера и страница администратора.

Не закрывайте окно **Gender Party Server** до окончания игры.


## Подключение игроков

1. Компьютер администратора и телефоны гостей должны быть в одной Wi‑Fi-сети.
2. На странице администратора нажмите кнопку с иконкой QR-кода.
3. Покажите QR-код гостям или отправьте ссылку вида:

```text
http://192.168.1.25:5000/
```

4. Каждый игрок открывает ссылку на своём телефоне и вводит уникальный никнейм.

Администратор открывает секретную страницу только на управляющем компьютере:

```text
http://127.0.0.1:5000/ЗНАЧЕНИЕ_ADMIN_SECRET_PATH
```

## Как проходит игра

### Обычные вопросы

Администратор нажимает карточку стоимости. Вопрос появляется у игроков. Каждый игрок выбирает или вводит ответ.

После повторного нажатия на карточку вопрос закрывается:

- правильный ответ: стоимость вопроса прибавляется;
- неправильный ответ: стоимость вопроса вычитается;
- ответ не отправлен: баллы не меняются.

### Аукцион

В аукционе участвуют только подключённые игроки с положительным количеством баллов.

- ставка не может быть больше текущего рейтинга игрока;
- после получения всех ставок выбирается максимальная;
- при равных ставках выигрывает более ранняя;
- на вопрос отвечает только победитель;
- при правильном ответе ставка прибавляется;
- при неправильном ответе ставка вычитается.

### Финальный раунд

Кнопка финального раунда становится доступной после использования всех 20 вопросов.

Игроки выбирают «Мальчик» или «Девочка». После раскрытия результата баллы угадавших игроков удваиваются.

### Секретный раунд

После раскрытия пола администратор повторно нажимает финальную карточку.

- каждый игрок может предложить только одно имя;
- игроки отдают свои баллы понравившимся именам;
- список сортируется по рейтингу от большего к меньшему.

### Сброс игры

Кнопка **Сбросить игру** удаляет:

- всех игроков;
- баллы;
- ответы;
- ставки;
- финальные голоса;
- предложенные имена;
- отметки использованных вопросов.

После сброса гости вводят никнеймы заново.

## Настройки `.env`

| Параметр | Пример | Назначение |
|---|---|---|
| `SECRET_KEY` | `long-random-string` | Подпись локальной сессии администратора |
| `ADMIN_SECRET_PATH` | `my-private-admin-page` | Секретная часть URL администратора |
| `ACTUAL_GENDER` | `boy` или `girl` | Правильный ответ финального раунда |
| `HOST` | `0.0.0.0` | Разрешает подключение устройств из локальной сети |
| `PORT` | `5000` | Порт сервера |
| `DEBUG` | `0` | Режим отладки; для игры оставьте `0` |

Файл `.env` внесён в `.gitignore` и не должен попадать в GitHub.

## Как сменить пол ребёнка

1. Остановите сервер, закрыв окно **Gender Party Server**.
2. Откройте `.env`.
3. Укажите одно из значений:

```env
ACTUAL_GENDER=boy
```

или:

```env
ACTUAL_GENDER=girl
```

4. Сохраните файл.
5. Снова запустите `start_game.bat`.
6. Перед новой игрой нажмите **Сбросить игру**.

Значения должны быть написаны строчными английскими буквами без кавычек.

## Как заменить вопросы

Вопросы находятся в файле:

```text
data/questions.json
```

Файл должен содержать JSON-массив из 20 объектов: по одному вопросу на каждую ячейку игрового поля.

### Фиксированная структура поля

В текущей версии поддерживаются ровно четыре категории:

```text
Беременность
Родители
Роды
Что это?
```

В каждой категории должны присутствовать вопросы стоимостью:

```text
100, 200, 300, 400, 500
```

Названия категорий и набор стоимостей нельзя менять только через JSON: для этого потребуется изменение Python, JavaScript и CSS.

### Вопрос с выбором ответа

```json
{
  "id": "pregnancy_100",
  "category": "Беременность",
  "points": 100,
  "type": "choice",
  "question": "Сколько в среднем длится беременность?",
  "options": [
    "30 недель",
    "35 недель",
    "40 недель",
    "45 недель"
  ],
  "correct_answers": [
    "40 недель"
  ],
  "is_auction": false
}
```

Для `type: "choice"`:

- `options` должен содержать варианты кнопок;
- правильный вариант должен присутствовать в `correct_answers`;
- текст варианта и правильного ответа лучше писать одинаково.

### Вопрос с текстовым ответом

```json
{
  "id": "pregnancy_400",
  "category": "Беременность",
  "points": 400,
  "type": "text",
  "question": "Какой временный орган формируется во время беременности?",
  "options": [],
  "correct_answers": [
    "плацента",
    "детское место"
  ],
  "is_auction": false
}
```

В `correct_answers` перечисляйте распространённые варианты написания.

При проверке ответа программа:

- не различает прописные и строчные буквы;
- считает `ё` и `е` одинаковыми;
- игнорирует знаки препинания и лишние пробелы;
- не исправляет смысловые ошибки и опечатки автоматически.

Например, для ответа «эпидуральная анестезия» полезно добавить:

```json
"correct_answers": [
  "эпидуральная анестезия",
  "эпидуриальная анестезия",
  "эпидуральная",
  "эпидуралка"
]
```

### Фото-карточка

1. Поместите изображение в папку `data`.
2. Используйте формат `.jpg` или `.jpeg`.
3. Укажите только имя файла, без полного пути:

```json
{
  "id": "parents_500",
  "category": "Родители",
  "points": 500,
  "type": "text",
  "question": "Кто изображён на фотографии?",
  "image": "family-photo.jpg",
  "options": [],
  "correct_answers": [
    "мама",
    "будущая мама"
  ],
  "is_auction": true
}
```

Рекомендуемые параметры:

- JPG/JPEG;
- до 1600×1200 пикселей;
- до 1 МБ;
- горизонтальная или квадратная ориентация;
- имя файла латиницей, без пробелов.

Изображение автоматически масштабируется внутри карточки и не выходит за её границы.

### Аукционный вопрос

Чтобы сделать вопрос аукционным:

```json
"is_auction": true
```

Обычно достаточно одного аукционного вопроса, но технически их может быть несколько.

### Правила для `id`

- каждый `id` должен быть уникальным;
- используйте латинские буквы, цифры и `_`;
- при редактировании существующего вопроса лучше сохранять прежний `id`;
- не используйте один `id` для двух вопросов.

### Проверка JSON перед запуском

Windows:

```powershell
.\.venv\Scripts\python.exe -m json.tool data\questions.json > $null
```

macOS/Linux:

```bash
.venv/bin/python -m json.tool data/questions.json >/dev/null
```

Если команда завершается без сообщения, синтаксис JSON корректен.

При запуске приложение дополнительно проверяет:

- уникальность `id`;
- наличие всех 20 ячеек;
- допустимые категории и стоимости;
- тип вопроса;
- наличие правильных ответов;
- существование указанных JPG-файлов.

После замены вопросов остановите сервер, перезапустите его и начните новую игру через **Сбросить игру**.

## Решение частых проблем

### Телефон не открывает ссылку

Проверьте:

- телефон и компьютер находятся в одной сети;
- используется адрес `192.168...`, а не `127.0.0.1`;
- на телефоне отключён VPN;
- сеть не является гостевой с изоляцией клиентов;
- Windows разрешил Python доступ к частным сетям;
- порт из `.env` не заблокирован брандмауэром.

### Появилось окно брандмауэра Windows

Разрешите доступ только для **частных сетей**. Для публичных сетей разрешение не требуется.

### Порт уже занят

Измените `.env`:

```env
PORT=5050
```

Перезапустите сервер.

### Картинка не показывается

Проверьте:

- файл находится непосредственно в `data`;
- имя в JSON совпадает полностью, включая регистр;
- расширение `.jpg` или `.jpeg`;
- в имени нет лишнего пути;
- JSON сохранён в кодировке UTF-8.

### После изменения вопросов отображаются старые данные

Остановите сервер, удалите локальные файлы:

```text
instance/game.sqlite
instance/game.sqlite-shm
instance/game.sqlite-wal
```

Затем снова запустите игру. Все результаты будут сброшены.

### Pylance не видит Flask или qrcode

В Visual Studio Code выберите интерпретатор:

```text
.venv\Scripts\python.exe
```

Команда: **Python: Select Interpreter**.

### Страница открывается, но статус показывает потерю соединения

Проверьте доступ к интернету: клиентская библиотека Socket.IO загружается с CDN. Затем обновите страницу сочетанием `Ctrl+F5`.
