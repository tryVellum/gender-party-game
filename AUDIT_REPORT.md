# Итоговая проверка публичного релиза

Дата проверки: 16 июля 2026 года.

## Исправленные проблемы

| Область | Было | Стало |
|---|---|---|
| Пол ребёнка | Ответ `boy` был жёстко задан в Python и JavaScript | Настройка `ACTUAL_GENDER=boy/girl` в `.env`, динамический текст и оформление финала |
| Адрес администратора | `ADMIN_SECRET_PATH` существовал в конфигурации, но маршрут был зашит как `/admin-secret-2026` | Маршрут формируется из `.env` |
| Защита административных действий | Секретный URL скрывал страницу, но административные API можно было вызвать напрямую | API администратора доступны только после открытия секретной страницы и установки сессии |
| Windows-запуск | `.bat` был привязан к папке конкретного пользователя | Скрипты используют собственную папку `%~dp0` и работают после распаковки в любое место |
| Локальные секреты | Настройки были значениями по умолчанию в коде | `.env` исключён из Git, `init_env.py` генерирует случайный ключ и секретный путь |
| Зависимости | Старые версии и Eventlet, несовместимый с Python 3.13 в исходной версии | Flask 3.1.3, Flask-SocketIO 5.6.1, threading-режим без Eventlet |
| Вопросы | Ошибки структуры JSON или отсутствующие фотографии обнаруживались поздно | При запуске проверяются 20 ячеек, уникальность ID, типы, ответы и наличие JPG |
| Замена набора | Удалённые из JSON вопросы могли оставаться в SQLite | Устаревшие записи синхронизируются при запуске |
| Публичные данные | Набор содержал личные имена, факты и ссылку на семейное фото | В публичный комплект включён нейтральный демонстрационный набор |
| Контроль качества | Автоматических проверок не было | Добавлены Ruff, Pytest и GitHub Actions для Python 3.11–3.13 |

## Выполненные проверки

- компиляция `app.py`, `config.py`, `database.py`;
- Ruff lint без ошибок;
- Ruff format check;
- проверка синтаксиса `admin.js` и `player.js` через Node.js;
- проверка JSON через стандартный `json.tool`;
- создание и повторный запуск генератора `.env`;
- запуск реального HTTP-сервера;
- загрузка страницы игрока;
- загрузка секретной страницы администратора;
- проверка запрета административного API без сессии;
- загрузка игрового поля после авторизации;
- регистрация двух игроков;
- Socket.IO-подключение и идентификация игрока;
- обычный вопрос и начисление баллов;
- аукцион с одним допустимым участником;
- текстовый ответ победителя аукциона;
- финальный раунд с правильным ответом `girl`;
- секретный раунд и ограничение «один игрок — одно имя»;
- полный сброс игры и удаление игроков.

Локальный автоматический тест завершён успешно на Python 3.13.5.

## Известные ограничения

1. Игровое поле фиксировано: 4 категории × 5 стоимостей. Для другого размера потребуется изменение интерфейса и серверной логики.
2. Клиент Socket.IO загружается с внешнего CDN. Для первого открытия страниц нужен интернет.
3. Проект предназначен для доверенной локальной сети, а не для размещения в открытом интернете.
4. Автоматические тесты проверяют серверную логику и Socket.IO-события, но не заменяют финальную репетицию на реальных телефонах, телевизоре или проекторе.
5. Медицинские и справочные формулировки демонстрационных вопросов следует воспринимать как пример структуры и проверять перед мероприятием.

## Рекомендация перед мероприятием

За день до игры проведите репетицию минимум с двумя телефонами:

1. запустите сервер;
2. подключитесь по QR-коду;
3. проверьте один обычный вопрос;
4. проверьте аукцион;
5. временно отметьте вопросы использованными или используйте тестовую копию базы для проверки финала;
6. проверьте показ результата на основном экране;
7. нажмите полный сброс перед настоящей игрой.
